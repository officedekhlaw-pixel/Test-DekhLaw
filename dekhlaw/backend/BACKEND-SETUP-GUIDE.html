2
Install Node Packages
Download all backend dependencies

Open your terminal, navigate into the backend folder, and run:

cd dekhlaw-backend
npm install

This will download all required packages into a node_modules/ folder. It takes 1–2 minutes. When done you will see a package-lock.json file created.

✓ What gets installed: Express (server), Mongoose (MongoDB), bcryptjs (password hashing), jsonwebtoken (login sessions), multer (file uploads), nodemailer (emails), express-validator (form validation), express-rate-limit (security).

3
Create Your .env File
Configure your database, email and secrets

Copy the template file and rename it:

cp .env.example .env

Now open .env in VS Code and fill in the values:

# ── Server ────────────────────────────────────────────────
PORT=5000
NODE_ENV=development

# ── MongoDB (choose Option A or B below) ──────────────────
MONGO_URI=mongodb://localhost:27017/dekhlaw

# ── JWT Secret (make this long and random) ────────────────
JWT_SECRET=dekhlaw_super_secret_key_change_this_abc123xyz
JWT_EXPIRES_IN=7d

# ── Email (Gmail SMTP) ────────────────────────────────────
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your@gmail.com
EMAIL_PASS=your_16_char_app_password
EMAIL_FROM=DekhLaw <noreply@dekhlaw.com>

# ── Frontend URL (for security) ───────────────────────────
CLIENT_URL=http://localhost:3000

MongoDB — 2 options:

Option A — Local MongoDB (free, on your computer):
Keep MONGO_URI=mongodb://localhost:27017/dekhlaw
Make sure MongoDB is running: open a new terminal and run mongod

Option B — MongoDB Atlas (free cloud, recommended for production):
1. Go to mongodb.com/atlas → Create free account
2. Create a free cluster → Click "Connect" → "Connect your application"
3. Copy the connection string. It looks like:
mongodb+srv://username:password@cluster0.abc.mongodb.net/dekhlaw
4. Paste it as your MONGO_URI in .env

Gmail App Password (for sending emails):

1. Go to your Google Account → Security → 2-Step Verification (enable it)
2. Search "App Passwords" → Select "Mail" → "Other" → type "DekhLaw"
3. Google gives you a 16-character password → paste it as EMAIL_PASS

4
Start the Backend Server
Run your API locally
# Development mode (auto-restarts when you edit files)
npm run dev

# OR production mode
npm start

You should see this in your terminal:

✅ MongoDB connected
🚀 DekhLaw API running on port 5000

Test it by opening your browser and going to:

http://localhost:5000/api/health

You should see:

{ "success": true, "message": "DekhLaw API is running" }

✓ Backend is running! The API is now live at http://localhost:5000

Open dekhlaw-updated/js/api.js and confirm line 1 matches your server address:

const API_BASE = 'http://localhost:5000/api'; // ← this should match your PORT in .env

6
Create Your Admin Account
Set up the admin user to manage the platform

First register a normal user through the website. Then open MongoDB Compass (or the Atlas dashboard) and manually change their role to admin:

# In MongoDB Compass or Atlas — find your user and update:
{ "role": "admin" }

Or use the MongoDB shell:

mongosh
use dekhlaw
db.users.updateOne({ email: "your@email.com" }, { $set: { role: "admin" } })

Now when you log in with that account and include the JWT token, you can access admin-only endpoints.

8
Deploy to Production (Go Live)
Host your backend so anyone can access it

Recommended free hosting options:

Option A — Railway (easiest, 1 click):
1. Go to railway.app → Sign in with GitHub
2. "New Project" → "Deploy from GitHub repo" → select your backend folder
3. Add your .env variables in the Railway dashboard under "Variables"
4. Railway gives you a URL like: https://dekhlaw-backend.up.railway.app
5. Update API_BASE in your js/api.js to this URL

Option B — Render (also free):
1. Go to render.com → New → Web Service
2. Connect your GitHub repo, set Build Command: npm install, Start Command: node server.js
3. Add environment variables → Deploy

After deploying, update this one line in js/api.js:

const API_BASE = 'https://your-live-backend-url.com/api';
